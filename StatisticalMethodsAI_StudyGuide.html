📌 1.1 — Population vs. Sample
Definition
Population: The complete set of all elements/individuals being studied.
Sample: A subset of the population selected for analysis.
Parameter: A numerical value describing a population (e.g., μ, σ).
Statistic: A numerical value describing a sample (e.g., x̄, s).

Descriptive Statistics: Summarizes data (mean, charts).
Inferential Statistics: Uses sample to make predictions about population (hypothesis tests, CI).
Simple explanation
Imagine you want to know the average marks of all CSE students in India. Testing ALL of them = population study (impossible). Testing 1000 random students = sample. The average of those 1000 is a "statistic" that estimates the population "parameter."
📌 1.2 — Types of Data
Qualitative (Categorical): Non-numeric
  • Nominal: no order (colors, blood types)
  • Ordinal: ordered (ranks, grades A/B/C)

Quantitative (Numerical): Numeric
  • Discrete: countable (number of bugs, students)
  • Continuous: measurable (height, temperature, time)
Key question: "Can you do meaningful math with it?" Qualitative = No (Red + Blue = ?). Quantitative = Yes (marks, age, salary).
📌 1.3 — Probability Fundamentals
Definition
Basic Probability: P(A) = favourable outcomes / total outcomes  (0 ≤ P(A) ≤ 1)

Addition Rule: P(A∪B) = P(A) + P(B) − P(A∩B)

Conditional Probability: P(A|B) = P(A∩B) / P(B)

Bayes' Theorem: P(A|B) = [P(B|A) · P(A)] / P(B)
Solved Example — Bayes' Theorem (Spam Filter)

60% of emails are spam. Of spam, 90% contain "free." Of non-spam, 10% contain "free." An email contains "free." Find P(spam|free).

Given: P(Spam)=0.6, P(¬Spam)=0.4, P(free|Spam)=0.9, P(free|¬Spam)=0.1

1
P(free) = 0.9×0.6 + 0.1×0.4 = 0.54 + 0.04 = 0.58
2
P(Spam|free) = (0.9×0.6)/0.58 = 0.54/0.58 ≈ 0.931 = 93.1%

→ 93.1% chance the email is spam. This is exactly how AI spam filters work!

📌 1.4 — Random Variables & Probability Distributions
Random Variable: Variable whose value is determined by a random event.
Discrete: PMF → P(X=x)  |  Continuous: PDF → f(x), area under curve = 1

Normal: f(x) = (1/σ√2π)·e^(−(x−μ)²/2σ²)
Binomial: P(X=k) = C(n,k)·p^k·(1−p)^(n−k)
Poisson: P(X=k) = λ^k·e^(−λ)/k!
Exponential: f(x) = λe^(−λx)
Normal distribution is everywhere: heights, exam marks, AI model errors. The 68-95-99.7 Rule: 68% of data falls within 1σ of mean, 95% within 2σ, 99.7% within 3σ.
Solved Example — Binomial Distribution

An AI model has 80% accuracy. Tested on 5 samples, find P(exactly 4 correct).

Given: n=5, k=4, p=0.8, (1−p)=0.2

1
C(5,4) = 5!/(4!·1!) = 5
2
P(X=4) = 5 × (0.8)⁴ × (0.2)¹ = 5 × 0.4096 × 0.2 = 0.4096 ≈ 41%
📌 1.5 — Hypothesis Testing
H₀ (Null): Default assumption (e.g., "no improvement")
H₁ (Alternative): What we want to prove
p-value: P(observing our result | H₀ is true)
Significance level α: Usually 0.05. Reject H₀ if p-value < α

Type I Error (α): Reject H₀ when true (False Positive)
Type II Error (β): Fail to reject H₀ when false (False Negative)
Like a courtroom: H₀ = "innocent." We need strong evidence (low p-value) to declare "guilty." If p < 0.05, evidence is strong enough → reject H₀.
Confidence Interval: CI = x̄ ± z*(σ/√n)
z* = 1.645 (90%)  |  z* = 1.96 (95%)  |  z* = 2.576 (99%)
Solved Example — 95% Confidence Interval

Sample of 100 students: mean score = 75, σ = 10. Find 95% CI.

1
SE = σ/√n = 10/√100 = 1
2
Margin of Error = 1.96 × 1 = 1.96
3
CI = 75 ± 1.96 = (73.04, 76.96)

→ We are 95% confident the true mean lies between 73.04 and 76.96.

📌 1.6 — EDA: Exploratory Data Analysis
Central Tendency: Mean (x̄=Σx/n), Median, Mode
Spread: Range, Variance s²=Σ(xᵢ−x̄)²/(n−1), Std Dev s=√s², IQR
Pearson Correlation: r = Σ[(xᵢ−x̄)(yᵢ−ȳ)] / √[Σ(xᵢ−x̄)²·Σ(yᵢ−ȳ)²]
r=1: perfect positive  |  r=−1: perfect negative  |  r=0: no linear relation

Visualizations: Histogram (shape) · Box plot (outliers) · Scatter plot (relationship) · Correlation matrix (all pairs)
📌 2.1 — Simple Linear Regression (SLR)
Model: Ŷ = β₀ + β₁X  (β₀ = intercept, β₁ = slope)

Least Squares Estimates:
β₁ = Sxy / Sxx where Sxy = Σ(xᵢ−x̄)(yᵢ−ȳ), Sxx = Σ(xᵢ−x̄)²
β₀ = ȳ − β₁·x̄

Residual (error): eᵢ = yᵢ − Ŷᵢ
The "least squares" line minimizes the sum of squared vertical distances from each data point to the line. β₁ tells us: "For every 1-unit increase in X, Y changes by β₁ units on average."
Solved Example — Finding the Regression Line

Hours studied X: 1,2,3,4,5  |  Marks Y: 50,55,65,70,75. Find Ŷ.

1
x̄ = (1+2+3+4+5)/5 = 3  |  ȳ = (50+55+65+70+75)/5 = 63
2
Sxy = (−2)(−13)+(−1)(−8)+(0)(2)+(1)(7)+(2)(12) = 26+8+0+7+24 = 65
3
Sxx = 4+1+0+1+4 = 10
4
β₁ = 65/10 = 6.5  |  β₀ = 63 − 6.5×3 = 43.5
5
Ŷ = 43.5 + 6.5X  → For X=6: Ŷ = 43.5+39 = 82.5 marks
📌 2.2 — Regression Evaluation Metrics
MSE: MSE = (1/n)Σ(yᵢ−Ŷᵢ)²
RMSE: √MSE — in same units as Y
R² (Coefficient of Determination): R² = 1 − SSres/SStot
SSres = Σ(yᵢ−Ŷᵢ)²  |  SStot = Σ(yᵢ−ȳ)²
R²∈[0,1]: R²=1 perfect fit, R²=0 model explains nothing
R²=0.85 means "our model explains 85% of the variation in Y." The remaining 15% is noise. Higher R² is better, but watch out for overfitting!
Multiple Regression: Ŷ = β₀ + β₁X₁ + β₂X₂ + ... + βₖXₖ

Logistic Regression (classification): P(Y=1) = 1/(1+e^−(β₀+β₁X))
Outputs probability 0–1 via sigmoid. Used for spam/fraud/disease detection.
📌 2.3 — Correlation Analysis
Pearson r: r = Sxy / √(Sxx·Syy)  (−1 ≤ r ≤ 1)
Coefficient of Determination: r² = proportion of Y's variability explained by X
Partial Correlation: Correlation between two variables after controlling for others
Multiple Correlation (R): Correlation between Y and all predictors combined
Solved Example — Correlation Coefficient

From previous example: Sxy=65, Sxx=10, Syy=350. Find r and R².

1
r = 65/√(10×350) = 65/√3500 = 65/59.16 ≈ 0.979
2
R² = 0.979² ≈ 0.958

→ Very strong positive correlation. Hours studied explains 95.8% of the variation in marks!

📌 2.4 — Non-linear Regression & Regularization
Polynomial Regression: Ŷ = β₀ + β₁X + β₂X² + ... + βₖXᵏ

Regularization (prevent overfitting):
  • Ridge (L2): minimize MSE + λΣβ²
  • Lasso (L1): minimize MSE + λΣ|β| (can zero out coefficients = feature selection)
When data follows a curve (not a straight line), use polynomial regression. Regularization prevents the model from memorizing training data and failing on new data (overfitting).
📌 2.5 — Time Series Analysis
Components:
  • Trend (T): Long-term direction
  • Seasonality (S): Regular periodic patterns
  • Cyclical (C): Irregular longer-term fluctuations
  • Irregular/Residual (I): Random noise

Additive model: Y = T + S + C + I
Multiplicative model: Y = T × S × C × I

Moving Average: MAₜ = (Yₜ + Yₜ₋₁ + ... + Yₜ₋ₖ₊₁) / k
Decomposing data into trend + seasonality + noise lets us model each part separately. Used in stock prediction, demand forecasting, Netflix recommendations, and weather prediction.
📌 3.1 — Estimators and Their Properties
Estimator (θ̂): A formula used to estimate a population parameter from sample data.
Point estimator: Single value (e.g., x̄ estimates μ)
Interval estimator: Range of values (confidence interval)

Properties (UCES):
  • Unbiasedness: E(θ̂) = θ. Bias = E(θ̂) − θ
  • Consistency: As n→∞, θ̂ → θ (converges to true value)
  • Efficiency: Minimum variance among all unbiased estimators
  • Sufficiency: Captures all information about θ in the sample
Cramér-Rao Lower Bound (CRLB): The minimum variance any unbiased estimator can achieve:
Var(θ̂) ≥ 1/I(θ)
where I(θ) = Fisher Information = −E[∂²/∂θ² ln f(X;θ)]
If Var(θ̂) = CRLB → estimator is called efficient
Dart analogy: Unbiased = average landing on bullseye | Consistent = more throws, closer to center | Efficient = throws tightly clustered (low variance). You want all three!
Solved Example — Proving Unbiasedness of x̄

Show that the sample mean x̄ is an unbiased estimator of μ.

1
E(x̄) = E[(x₁+x₂+...+xₙ)/n] = (1/n)·E(x₁+...+xₙ)
2
= (1/n)·[E(x₁)+...+E(xₙ)] = (1/n)·[μ+μ+...+μ] = (1/n)·nμ
3
= μ → E(x̄) = μ, so x̄ is unbiased ✓

Note: Sample variance with (n−1) denominator is unbiased. With n denominator it's biased.

📌 3.2 — Method of Moments (MOM)
Idea: Set population moments = sample moments, then solve for parameters.
k-th population moment: μₖ' = E(Xᵏ)
k-th sample moment: mₖ = (1/n)Σxᵢᵏ
Equate and solve for unknown parameters.
Solved Example — MOM for Normal Distribution

For X ~ N(μ, σ²), find MOM estimators of μ and σ².

1
1st moment: E(X)=μ → set equal to m₁=x̄ → μ̂ = x̄
2
2nd moment: E(X²) = σ²+μ² → set equal to m₂=(1/n)Σxᵢ²
3
σ² = E(X²)−μ² → σ̂² = (1/n)Σ(xᵢ−x̄)²

Note: MOM gives biased σ̂² (divides by n, not n−1).

📌 3.3 — Maximum Likelihood Estimation (MLE)
Likelihood: L(θ) = Π f(xᵢ;θ) — joint probability of observing our data given θ
Log-Likelihood: ℓ(θ) = Σ ln f(xᵢ;θ)
MLE: ∂ℓ(θ)/∂θ = 0 → solve for θ̂

MLE Properties: Consistent · Asymptotically efficient (reaches CRLB) · Asymptotically normal · Invariant: if θ̂ is MLE of θ, then g(θ̂) is MLE of g(θ)
MLE asks: "Given what I observed, what parameter value makes this observation MOST LIKELY?" Like a detective: "Given this evidence, what scenario best explains it?"
Solved Example — MLE for Exponential Distribution

X₁,...,Xₙ ~ Exponential(λ), f(x;λ)=λe^(−λx). Find MLE of λ.

1
L(λ) = Π λe^(−λxᵢ) = λⁿ·e^(−λΣxᵢ)
2
ℓ(λ) = n·ln(λ) − λΣxᵢ
3
∂ℓ/∂λ = n/λ − Σxᵢ = 0
4
Solve: λ̂ = n/Σxᵢ = 1/x̄

→ If average waiting time is x̄, the rate = 1/x̄. Intuitive!

📌 3.4 — Law of Large Numbers & Central Limit Theorem
Weak LLN: P(|x̄−μ| > ε) → 0 as n→∞ (x̄ converges in probability to μ)
Strong LLN: x̄ → μ almost surely (stronger convergence)

Central Limit Theorem (CLT):
Regardless of population distribution, x̄ becomes normal as n→∞:
x̄ ~ N(μ, σ²/n) for large n (rule of thumb: n ≥ 30)
Z = (x̄ − μ) / (σ/√n) ~ N(0,1)
CLT is the MOST IMPORTANT theorem in statistics! Even if population is skewed/weird, the distribution of sample means becomes bell-shaped with enough samples. This is why so many tests assume normality!
Solved Example — CLT in Action

Server response times ~ Exponential, mean=200ms, σ=200ms. Sample n=100. Find P(x̄ < 210).

1
By CLT: x̄ ~ N(200, 200²/100) = N(200, 400)
2
SE = 200/√100 = 20
3
Z = (210−200)/20 = 0.5
4
P(x̄ < 210) = P(Z < 0.5) = 0.6915 ≈ 69.15%
📌 3.5 — Asymptotic Properties
Asymptotic Normality: √n(θ̂−θ) → N(0, σ²) as n→∞
Asymptotic Efficiency: Achieves CRLB in large samples
Asymptotic Variance: The σ² in the limiting normal distribution

Asymptotic Relative Efficiency: ARE(θ̂₁,θ̂₂) = AVar(θ̂₂)/AVar(θ̂₁)
ARE > 1 → θ̂₁ is more efficient than θ̂₂
Probability
P(A) = fav/total
P(A∪B) = P(A)+P(B)−P(A∩B)
P(A|B) = P(A∩B)/P(B)
Bayes: P(A|B) = P(B|A)P(A)/P(B)
Distributions
Binomial: C(n,k)·pᵏ·(1−p)ⁿ⁻ᵏ
Poisson: λᵏ·e⁻λ/k!
Normal: (1/σ√2π)e^−(x−μ)²/2σ²
Exponential: λe^(−λx)
Descriptive Stats
Mean: x̄ = Σxᵢ/n
Variance: s²=Σ(xᵢ−x̄)²/(n−1)
Std Dev: s = √s²
z-score: z = (x−μ)/σ
Confidence Intervals
CI = x̄ ± z*(σ/√n)
SE = σ/√n
90%: z*=1.645
95%: z*=1.96
99%: z*=2.576
Regression
Ŷ = β₀ + β₁X
β₁ = Sxy/Sxx
β₀ = ȳ − β₁x̄
Sxy=Σ(xᵢ−x̄)(yᵢ−ȳ)
Sxx=Σ(xᵢ−x̄)²
Regression Metrics
MSE = (1/n)Σ(yᵢ−Ŷᵢ)²
RMSE = √MSE
R²=1−SSres/SStot
r=Sxy/√(Sxx·Syy)
Estimation
Bias = E(θ̂)−θ
Unbiased: E(θ̂)=θ
CRLB: Var(θ̂)≥1/I(θ)
Log-L: ℓ=Σln f(xᵢ;θ)
MLE: dℓ/dθ=0
CLT & LLN
CLT: x̄ ~ N(μ, σ²/n)
SE = σ/√n
Z = (x̄−μ)/(σ/√n)
LLN: x̄ →ₚ μ as n→∞
MLE Results
Normal μ: μ̂=x̄
Normal σ²: σ̂²=(1/n)Σ(xᵢ−x̄)²
Exponential λ: λ̂=1/x̄
Binomial p: p̂=x̄/n
🧠 Properties of Good Estimators
Mnemonic: "UCES" → "Uncle CES loves statistics"
U = Unbiasedness · C = Consistency · E = Efficiency · S = Sufficiency
Dart analogy: Unbiased=centered on bullseye | Consistent=gets better with more throws | Efficient=throws are tightly clustered (low variance)
🧠 Hypothesis Testing Steps
"SHAFT" Method:
S = State H₀ and H₁ | H = Set significance level α | A = Assume distribution | F = Find test statistic | T = Test (compare p-value to α, reject if p < α)
🧠 z* Critical Values (must memorize!)
"One-Two-Three → Ninety-NinetyFive-NinetyNine"
z*≈1.645 → 90% CI | z*≈1.96 → 95% CI | z*≈2.576 → 99% CI

Shortcut: "1.96 ≈ 2" → 95% CI ≈ x̄ ± 2·SE (quick mental math!)
🧠 Normal Distribution 68-95-99.7 Rule
"All the sixes and nines:"
μ ± 1σ → 68.27% of data
μ ± 2σ → 95.45% of data
μ ± 3σ → 99.73% of data

Trick: "Almost everything (99.7%) lives within 3 standard deviations."
🧠 r vs. R²
r = direction + strength of linear relationship
R² = % of variance in Y explained by X
Always square r to get R². Example: r=0.9 → R²=0.81 → "81% of Y's variation explained."
🧠 MLE Recipe — "WLMD"
W = Write likelihood L(θ) = Π f(xᵢ;θ)
L = Log it: ℓ(θ) = Σ ln f(xᵢ;θ)
M = Maximize: set dℓ/dθ = 0, solve for θ̂
D = Declare θ̂ as your MLE
🧠 CLT vs. LLN — Key Difference
LLN: "More data → sample mean gets closer to true mean" (WHERE x̄ goes)
CLT: "Distribution of x̄ becomes normal, no matter the original distribution" (WHAT SHAPE it has)

LLN = Location | CLT = Shape
🧠 Regression β₁ in plain English
"Slope = Shared variation / X's own variation"
β₁ = Sxy/Sxx → "How much X&Y move together" ÷ "How much X moves alone"
Large positive Sxy → strong positive slope → X and Y rise together

Q: DDL vs DML?
DDL (Data Definition Language) changes structure — CREATE, ALTER, DROP. DML (Data Manipulation Language) changes data — INSERT, UPDATE, DELETE.
Q: TRUNCATE vs DELETE?
TRUNCATE removes all rows, can't use WHERE, faster, auto-commits. DELETE removes specific rows, supports WHERE, can rollback.

Q: INNER vs OUTER Join?
INNER returns only matching rows. OUTER (LEFT/RIGHT/FULL) includes non-matching rows with NULLs for missing data.
Q: Does MySQL support FULL OUTER JOIN?
No. MySQL doesn't have FULL OUTER JOIN. Use UNION of LEFT JOIN + RIGHT JOIN as workaround.

Q: View vs Table?
View is virtual — doesn't store data, it's a saved SELECT query. Table stores data physically on disk.
Q: Why use an Index?
Speeds up SELECT queries on frequently searched columns (B-Tree structure). Trade-off: slower INSERT/UPDATE because index must also be updated.
Q: What is a Cursor?
A mechanism to process query results row-by-row inside stored procedures. Used when you need complex per-row logic that can't be done with standard SQL.

WHERE filters individual rows BEFORE grouping. HAVING filters groups AFTER GROUP BY. You can't use aggregate functions in WHERE.

ANY, ALL, IN, EXISTS, UNION, INTERSECT

CO3
Aim: Implement queries using subquery operators and set operations.
Q: ANY vs ALL?
ANY: true if condition matches at least one subquery value. ALL: true only if condition matches every subquery value.
Q: IN vs EXISTS?
IN matches against a list of values. EXISTS checks if subquery returns any rows. EXISTS performs better on large datasets because it short-circuits.
Q: UNION vs UNION ALL?
UNION removes duplicate rows (slower). UNION ALL keeps all rows including duplicates (faster).

Triggers (Insert, Update, Delete)

CO4
Aim: Implement database triggers that execute automatically in response to INSERT, UPDATE, and DELETE operations.

6 Trigger Types

TimingINSERTUPDATEDELETE
BEFOREBEFORE INSERTBEFORE UPDATEBEFORE DELETE
AFTERAFTER INSERTAFTER UPDATEAFTER DELETE
Q: What is a Trigger?
A stored program that auto-executes when an INSERT, UPDATE, or DELETE event occurs on a table. Used for audit logging, enforcing business rules, maintaining derived data.
Q: BEFORE vs AFTER?
BEFORE: runs before the data change — can modify incoming data (NEW). AFTER: runs after data is committed — good for logging/audit trails.
Q: NEW vs OLD?
NEW = incoming/new data (available in INSERT & UPDATE). OLD = existing/current data (available in UPDATE & DELETE). INSERT has no OLD, DELETE has no NEW.

Top 15 Viva Questions

1. DDL vs DML?
DDL modifies structure (CREATE, ALTER, DROP). DML modifies data (INSERT, UPDATE, DELETE).
2. DELETE vs TRUNCATE vs DROP?
DELETE: row-by-row, rollbackable, uses WHERE. TRUNCATE: all rows, no rollback, no WHERE. DROP: removes table entirely.
3. PRIMARY KEY vs UNIQUE?
PK: one per table, no NULLs. UNIQUE: multiple per table, allows one NULL.
4. WHERE vs HAVING?
WHERE filters rows BEFORE GROUP BY. HAVING filters groups AFTER GROUP BY.
5. INNER vs LEFT vs RIGHT JOIN?
INNER: matching rows only. LEFT: all left + matched right (NULL if no match). RIGHT: all right + matched left.
6. View vs Table?
View is virtual (saved query, no storage). Table stores data physically.
7. Why use an Index?
Speeds up SELECT (B-Tree lookup). Slows INSERT/UPDATE. Use on frequently searched columns.
8. ANY vs ALL?
ANY: true if ≥1 subquery value matches. ALL: true only if every subquery value matches.
9. IN vs EXISTS?
IN: matches a list. EXISTS: checks if subquery returns rows. EXISTS is faster on large datasets.
10. UNION vs UNION ALL?
UNION removes duplicates (slower). UNION ALL keeps all rows (faster).
11. What is a Trigger?
Auto-executing stored program fired on INSERT/UPDATE/DELETE events.
12. BEFORE vs AFTER trigger?
BEFORE can modify incoming data. AFTER is for logging/audit after data is committed.
13. NEW vs OLD in triggers?
NEW = incoming data (INSERT/UPDATE). OLD = existing data (UPDATE/DELETE).
14. What is a Cursor?
Row-by-row processing of query results inside stored procedures.
15. COMMIT vs ROLLBACK?
COMMIT saves permanently. ROLLBACK undoes changes to last commit point.
